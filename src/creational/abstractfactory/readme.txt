Назначение:
Шаблон используется для создания множества взаимосвязанных объектов, без жёсткой привязки к конкретным классам.

1.) Все фабрики инкапсулируют создание объектов.
2.) Simple Factory не является полноценным паттерном, но обеспечивает простой механизм изоляции клиентов от
конкретных классов.
3.) Factory Method снован на наследовании: создание объектов делегируется сабклассам, реализующим factory method для
создания объектов.
4.) Abstract Factory основан на композиции: создание объектов реализуется в методе, доступ к которому осуществляется
через интерфейс фабрики.
5.) Все фабричные паттерны обеспечивают слабую связанность за счет сокращения зависимости приложения от конкретных
классов.
6.) Задача Factory Method - перемещение создания экземпляров в сабклассы.
7.) Задача Abstract Factory - создание семейств взаимосвязанных объектов без зависимости от их конкретных классов.
