Назначение:
Шаблон используется для создания единственного экземпляра класса и создания глобальной точки доступа к
этому классу.

1.) Паттерн Singleton гарантирует, что в приложении существует не более одного экземпляра данного класса.
2.) Паттерн предоставляет глобальную точку доступак этому экземпляру.
3.) Реализация паттерна использует приватный конструктор и статический метод в связке с статической переменной.
4.) Производительности системы чувствительна к реализации данного паттерна.
5.) Будьте внимательны при использовании ClassLoader'ов, они могут привести к созданию нескольких экземпляров класса.

Паттерн привносит в программу глобальное состояние, чем сильно затрудняет юнит-тестирование.

Классическая реализация паттерна не проверяет, существует ли один экземпляр на JVM, она лишь удостоверяется, что
существует один экземпляр на classloader, поэтому если существует несколько загрузчиков класса или приложение работает
на сервере, где может быть запущено несколько экземпляров приложения в разных classloader, то единственность инстанса
не может быть обеспечена на 100%.

Единственность создания класса обеспечивается только приватным конструктором и не защищает от создания объектов через
сериализацию/десериализацию и reflection.
